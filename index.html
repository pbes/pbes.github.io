    Creating CI/CD pipeline on CentOS

    Creating CI/CD pipeline on CentOS Last year I was working for a company and we had a project where I created a CI/CD pipeline for them. The leader of their IT operations really liked the result and the detailed documentation, but he left that company and contacted me on LinkedIn whether I could reproduce it for him for future reference. If you follow along with this post, you’ll be able to install a CI/CD pipeline running on CentOS 7 for enterprise Java applications with the following items: WildFly: to have a running TEST system on the same machine GitLab: to store source codes locally Jenkins: for continous integration Nexus: to store the built artifacts and any 3rd party Java libraries custom Maven configuration that can be used by all of the company’s developers to access own Nexus repository
